# Identity Schema

A machine-readable specification for personal identity — designed to govern how AI agents act on your behalf.

## The Problem

AI agents know what you've said. They don't know who you are.

Memory systems, knowledge brains, and RAG pipelines solve retrieval. They don't solve identity. An agent with access to 10,000 documents can find your words — but it can't consistently represent *you* across tools, contexts, and time.

Instructions are procedural: *what to do*. Identity is declarative: *who to be*.

## The Schema

Eight layers. Each one excavated through structured dialogue — not generated from documents.

| Layer | What it encodes |
|-------|----------------|
| **Identity** | The irreducible core that survives every pivot and rebrand |
| **Positioning** | Where you stand relative to everyone else |
| **Strategy** | The decision logic that connects identity to action |
| **Story** | The narrative excavated from what actually happened |
| **Voice** | The linguistic fingerprint no prompt can fake |
| **Worldview** | The convictions that shape decisions before you notice |
| **Principles** | The lines that make identity durable under pressure |
| **Evidence** | Proof. Not claims — artifacts. |

The output is a JSON specification that any LLM can interpret at runtime. It sits between the person and every tool that speaks on their behalf.

## Status

Active research and production use. The full schema includes voice dimension specifications, constraint hierarchies, evaluation personas, and runtime modulation logic. This repository contains the architectural overview.
